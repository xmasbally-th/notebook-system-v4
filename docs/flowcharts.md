## ภาพที่ 3.1 — กระบวนการยืมอุปกรณ์ (Lending Process)

```mermaid
flowchart LR
    A([เริ่ม]) --> B[ค้นหาอุปกรณ์ สถานะ ว่าง]
    B --> C[กรอกฟอร์มขอยืม]
    C --> D[ส่งคำขอ]
    D --> E{Staff พิจารณา}
    E -- อนุมัติ --> F[แจ้งผู้ใช้มารับเครื่อง]
    F --> G[รับอุปกรณ์ที่เคาน์เตอร์]
    G --> H([กำลังยืม])
    E -- ปฏิเสธ --> I([ปฏิเสธ + แจ้งเหตุผล])
```

## ภาพที่ 3.1b — กระบวนการจองล่วงหน้า (Reservation Process)

```mermaid
flowchart LR
    A([เริ่ม]) --> B[เลือกอุปกรณ์ที่ถูกยืมอยู่]
    B --> C[กรอกฟอร์มจองวันที่ต้องการ]

    F --> G[มารับอุปกรณ์]
    G --> H([กำลังยืม])
    E -- ปฏิเสธ --> I([จองถูกปฏิเสธ])
```

## ภาพที่ 3.2 — กระบวนการส่งคืนอุปกรณ์ (Return Process)

```mermaid
flowchart LR
    A([เริ่ม]) --> B[ผู้ใช้นำอุปกรณ์มาคืน]
    B --> C[Staff ค้นหาเลขคำขอยืม]
    C --> D{สภาพอุปกรณ์?}
    D -- สมบูรณ์ --> E[บันทึกคืนแบบปกติ]
    D -- ชำรุด/สูญหาย --> F[บันทึกความเสียหาย]
    E --> G[อุปกรณ์: ว่าง]
    F --> H[อุปกรณ์: ซ่อมบำรุง]
    G --> I[แจ้งเตือนผู้ใช้]
    H --> I
    I --> J[ส่งลิงก์แบบประเมิน]
    J --> K([สิ้นสุด])
```

## ภาพที่ 3.3 — กระบวนการบริหารจัดการสำหรับ Admin (Management Process)

```mermaid
flowchart LR
    Login([Admin เข้าสู่ระบบ]) --> Menu{เลือกเมนู}

    Menu --> A[จัดการผู้ใช้]
    A --> A1{อนุมัติบัญชี?}
    A1 -- อนุมัติ --> A2[กำหนด Role]
    A1 -- ปฏิเสธ --> A3[ลบบัญชี]

    Menu --> B[จัดการอุปกรณ์]
    B --> B1[เพิ่ม/แก้ไข/ลบ]

    Menu --> C[Dashboard]
    C --> C1[กราฟ + Log]

    Menu --> D[ตั้งค่าระบบ]
    D --> D1[Discord Webhook]
    D --> D2[Auto-Archive]
```

## ภาพที่ 3.4 — ระบบแจ้งเตือนอัตโนมัติ (Notification System)

```mermaid
flowchart LR
    T([Event เกิดขึ้น]) --> E{ประเภท Event}

    E -- ส่งคำขอยืม --> N1[แจ้ง Staff + Discord]
    E -- อนุมัติ/ปฏิเสธ --> N2[แจ้งผู้ใช้ในแอป]
    E -- ใกล้วันคืน --> N3[แจ้งล่วงหน้า 1 วัน]
    E -- อนุมัติบัญชีใหม่ --> N4[แจ้งยืนยันบัญชี]
    E -- คืนสำเร็จ --> N5[แจ้งยืนยันคืน]

    N1 --> D1[(Discord)]
    N2 --> D2[(In-App)]
    N3 --> D1
    N3 --> D2
    N4 --> D2
    N5 --> D2
```

## ภาพที่ 3.5 — ระบบ Support Chat (ช่องทางสื่อสาร)

```mermaid
flowchart LR
    A([Admin เปิด Ticket]) --> B[ส่งข้อความถึงผู้ใช้]
    B --> C[ผู้ใช้รับแจ้งเตือน]
    C --> D[เปิด Support Chat]
    D --> E[อ่าน + ตอบกลับ]
    E --> F[Admin รับข้อความ]
    F --> G{แก้ไขได้?}
    G -- ยังไม่ได้ --> B
    G -- แก้ไขแล้ว --> H[ปิด Ticket]
    H --> I([สิ้นสุด])
```
